## Why
- FrameView 目前仅提供文件树与目录面板，缺少快速插入媒体的入口，编辑时需要切换到媒体库查找，效率低。
- 需要在编辑侧边栏直接浏览所有图片、搜索定位并插入 Markdown，减少上下文切换。

## What Changes
- 在 FrameView 侧边栏新增 `mediaPanel`（与 `fileTree`、`tocPanel` 相同的面板机制），显示所有图片（按最近修改倒序、支持纵向滚动），缩略图列数随面板宽度自适应（不少于 1 列，缩略图宽度在 80px~160px 之间，超过则自动增加列），数据复用现有媒体接口（例如 `window.site.getAssets`）；每次用户展开 mediaPanel 时拉取最新资产，折叠后不缓存。
- 面板包含前端关键字搜索（仅匹配文件名，大小写不敏感，空输入不触发搜索）、匹配数量提示（显示“当前索引/总命中”）以及“上一条/下一条”跳转，高亮当前命中项以便辨识；搜索 UI 封装为可复用的 `SearchBar` 组件（输入框、命中计数、上下跳转按钮），其它需要文件名检索的界面（如 MediaLibrary、fileTree 等）也可复用。
- 点击缩略图打开放大预览对话框，提供“插入”与“关闭”操作；插入会生成 Markdown 图片语法并写入当前正文（若光标不在正文则追加到正文末尾，插入语句后另起一行）。Markdown `alt` 可暂留为空，路径根据当前文章 permalink 计算正确相对路径，始终指向 Hexo 生成后网站根目录下的 `images/` 资源（例如文章 permalink 为 `2025/test-article/` 时，图片 `images/foo.jpg` 插入为 `![](../../images/foo.jpg)`）。对话框内显示图片路径、尺寸、大小、创建时间等信息（紧凑排版），单击缩略图弹出预览、双击缩略图可直接插入并关闭对话框。
- 当用户通过编辑器内置的图片上传功能上传成功后，mediaPanel MUST 刷新媒体列表，将最新上传的图片显示在顶部。

## Impact
- 需要从现有媒体数据源获取图片列表（无需分页）并维护搜索/导航状态。
- 编辑器需要暴露插入接口（定位当前光标或正文末尾）；需确保不会影响标题等非正文输入区域。
